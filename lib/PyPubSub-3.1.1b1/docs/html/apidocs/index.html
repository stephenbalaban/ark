Versions¶

There are two supported APIs available in pubsub: the default API, and a legacy API named “version 1” or v1 for short.

The default API was developed after the original API was used in larger applications and found to provide inadequate support for message validation, debugging and documenting, all important considerations in larger projects. The various pages under the API section of this site document the default API.

The legacy API was developed as part of wxPython, where pubsub originated. The legacy API is activated by importing pubsub’s setupv1 module before the first pubsub import statement. See Legacy (v1) API for documentation on the v1 API of pubsub.

If you wish to transition an appication from using the legacy v1 API to using the default API, take a look at the Migrations page.
